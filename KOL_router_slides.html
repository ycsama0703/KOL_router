KOL ROUTER01 · current problem
02 / 11
现状问题是什么

Agent 不是缺社媒数据，
是不会高效选择早期信息

overload
候选叙事太多
一个 symbol-day 里会出现大量看起来合理的 KOL 叙事主题，不能全部进入 memory / RAG / LLM。
timing
热度确认太晚
等 follower、转发和二次传播出现后再判断，已经不是 early attention allocation。
llm bottleneck
Full LLM 不等于会用
LLM 看到更多候选，不一定能更好识别哪些早期叙事值得下游关注。
本文解决的是 first-stage routing：在叙事扩散前，决定哪些“由 KOL 首次提出的叙事主题”值得进入昂贵推理链路。
KOL ROUTER02 · discovery
03 / 11
我们发现了什么

我们发现的是“叙事起源关系”，不是 KOL 标签

核心判断是：这个新说法由这个历史角色的人在这个上下文中提出，是否值得进入下游推理。真实可用的信号不是“某个 KOL 永远好/坏”，而是新叙事、发起者历史角色、当前上下文三者的组合。
what structure
对象是“新叙事”，不是账号
我们路由的是事件中刚出现的叙事候选：某个说法第一次被谁提出，之后是否可能被其他 KOL 接续。
why useful
账号历史只是一个先验
历史源头分数只回答“这个发起者过去是否常站在扩散链前端”；它不能单独决定当前候选是否重要。
how it solves problem
真正信号来自组合
只有把“谁提出”与当前叙事的新颖性、可见度、时点结合，才形成可用于 agent 的低延迟路由信号。
新说法 × 发起者历史角色 × 当前上下文 → 是否进入下游推理

KOL ROUTER04 · method
05 / 11
有什么用

把“历史源头分数”变成低延迟前端 router

si = β′Xi + γOk(i) + δ₁(O·visibilityi) + δ₂(O·noveltyi)
  • 不是 KOL 排行榜　历史上常当源头，不代表每条都重要；要和当前 visibility / novelty / timing 结合。
  • 不是卖新 neural architecture　故意用最小线性 ridge；如果简单模型有效，价值更可能来自数据结构。
  • 适合 agent entry layer　zero prompt tokens · no online text-model inference · point-in-time · near-zero latency。

实验怎么做 · Dataset

主实验协议：只允许模型使用当时可见信息

PIT split: what models are allowed to know
历史源头分数只用 2020 前历史估计
model train: 2020-01-01 to 2021-06-01
validation: 2021-06-01 to 2022-06-01
ranking eval: 785 events / 16 symbols
candidate set used in experiments
semantic threshold = 0.55
origin window = first10
candidate = 事件中刚被某个 KOL 首次提出的叙事主题
events with at least two valid candidates enter ranking eval
label used for evaluation
future follower-weighted Reach
后续加入同一叙事主题的 KOL followers 总量的 log 变换；label 不进入 online router。
online information only
可用：最早提出者、原始文本、rank/time、visibility、novelty、历史源头分数
不可用：后续 follower 确认、完整扩散路径、事后热度。
大样本证明结构存在；PIT split 检验这个结构能否在真实在线场景中作为 router 使用。

结果 3 · 小实验

真实 agent 场景：先路由，再把短名单交给 LLM

setup
每个 symbol-day 给出 K=30 个候选早期叙事。Full LLM 直接读全部 30 个；OL-Origin 先用本地 router 选 b=10 或 b=20，再交给同一个 LLM 做 listwise reranking。
what this tests
指标是 fixed Kmax-oracle capture：在同一个 K=30 候选池里，最终选择捕获了 oracle 可达未来 reach 的多少。越高越好。

KOL ROUTER09 · takeaway
11 / 11
给老师的四个回答

结论：价值在 KOL 数据结构，不在模型容量

现状问题
Agent 面对高频社媒时，瓶颈是 first-stage attention allocation：哪些内容值得进入昂贵推理？
我们发现
KOL 流里有稳定、身份驱动、PIT 可用的历史源头分数，不是 follower / 时区 / 频率 / 新闻反应。
有什么用
用历史源头分数做低延迟结构化 router，先决定哪些叙事值得进入 LLM / memory / RAG。
实验怎么做
大样本 KOL panel，PIT split，强 benchmark families，ranking / calibration / latency metrics，主实验 + 消融 + listwise LLM routing。
如果大样本 KOL 结构用简单 ridge 就能有效路由，说明真正有价值的是结构本身，而不是隐藏模型容量。
